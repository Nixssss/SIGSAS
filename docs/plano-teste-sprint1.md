# Plano de Teste – Sprint 1

## 1. Objetivo
Validar o funcionamento do cadastro e login do sistema SIGSAS.

## 2. Escopo
- Cadastro de usuário
- Login usuário
- Integeração frontend/backend

## 3. Tipo de teste 
- Teste funcional
- Teste de API
- Teste de integração

## 4. Ambiente
- Backend local (FastApi)
- Frontend Local (React)
- Banco PostgreSQL local

## 5. Critérios de Aceitação
- Cadastrar usuário no banco
- Login retornar token válido
- Login inválido retornar erro adequado
